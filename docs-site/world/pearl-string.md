# The Pearl String Model

The Ember Galaxies universe is structured as a **linear chain** — not a random scatter of galaxies. This is by design.

## Structure

```
G1 ← G2 ← G3 ← ... ← G10 ← ... ← G100
```

| Level | Count | Details |
|-------|-------|---------|
| Galaxies | 100 | G1 to G100, arranged like a string of pearls |
| Systems | 300 | Per galaxy |
| Planets | 10-30 | Per system (normal distribution, avg 20) |
| Total planets | ~9 million | |

## Why Linear?

1. **Natural progression** — Players expand from their starting galaxy outward
2. **Newbie protection** — New players start in G10 (top, safe zone)
3. **Natural frontier** — Distance increases exponentially as you push further
4. **Endgame geography** — G1 (bottom) is veteran no-man's-land

## Travel Times

| Range | Time |
|-------|------|
| Within same system | Minutes to <1 hour |
| System to system | Hours |
| Galaxy to galaxy | Days to weeks (exponentially scaling) |

## Starting Position

New players begin in **G10** — the upper safe zone. This gives them room to expand without immediately encountering veteran players who have pushed toward G1.

## The Mid-Game Driver

As empires grow, resources become scarce in safe zones. The natural pressure is to migrate **downward** toward G1, where resources are richer but veterans lurk. This creates organic conflict.
